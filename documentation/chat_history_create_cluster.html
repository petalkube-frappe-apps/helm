What is Jetstack and the Process:

Jetstack is the company that created cert-manager, which is a fully open-source project. Yes, https://github.com/cert-manager/cert-manager is the official repository - it's the same project. The Helm repository (charts.jetstack.io) simply hosts the packaged Helm charts for easy installation. Cert-manager automates the process of obtaining, renewing, and managing SSL/TLS certificates from certificate authorities like Let's Encrypt. Once installed, you create a "ClusterIssuer" that connects to Let's Encrypt, then cert-manager automatically requests certificates for your domains and keeps them renewed. It integrates directly with Kubernetes Ingress resources, so when you add TLS configuration to your Ingress, cert-manager automatically handles the certificate. The entire process is automated - you just configure it once and it handles everything including 90-day renewals.

Clarification: There's only ONE cert-manager project. Both the GitHub repo and Jetstack Helm repo are the same tool. If you meant comparing cert-manager to other solutions, here's a comparison:

Feature cert-manager (Jetstack) Manual SSL Management Cloud Provider SSL (OCI Certs)
Pros Fully automated, free Let's Encrypt certs, auto-renewal, Kubernetes-native, open-source Full control, no dependencies Integrated with cloud, managed service
Cons Requires setup, uses cluster resources Manual renewal every 90 days, error-prone Costs money, vendor lock-in, less flexible

Do You Need to Redo This for ERPNext:

No, you do NOT need to reinstall cert-manager when switching to ERPNext - it's a one-time cluster-level installation. Cert-manager runs as a service in your cluster and watches ALL Ingress resources automatically. When you create a new Ingress for ERPNext (or update the existing one), cert-manager will automatically detect it and request certificates. The ClusterIssuer (Let's Encrypt configuration) you create now will work for any domain you add in the future. You'll simply create a new Ingress YAML file pointing to your ERPNext service instead of nginx-deployment, and cert-manager handles the SSL automatically. The only thing you'll change is the Ingress resource - the backend service name, and possibly the domain if different. Cert-manager will see the TLS configuration in your ERPNext Ingress and automatically provision certificates. This is the beauty of cert-manager - install once, use for all your applications.
